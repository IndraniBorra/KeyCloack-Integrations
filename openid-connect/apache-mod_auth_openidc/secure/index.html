You are authenticated via Keycloak OIDC!

User information is available via HTTP request headers set by mod_auth_openidc:

HeaderValue
REMOTE_USERSet to preferred_username claim
X-UserUsername from Keycloak token
X-EmailEmail from Keycloak token
OIDC_CLAIM_*All OIDC claims available as env vars

In a real app, a backend script (PHP, CGI, Python) would read these headers to identify the user.

Tokens

Access and ID tokens are managed server-side by mod_auth_openidc. To expose them, enable OIDCPassAccessToken On and OIDCPassIDTokenAs claims in oidc.conf. They will then be available as OIDC_access_token and OIDC_id_token_* environment variables for backend scripts.
